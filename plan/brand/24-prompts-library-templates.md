# 25 — Prompts Library Templates

## 25.1 Propósito

Spec detallado del **Prompts Library Markdown** — el deliverable #2, prompts pre-escritos customizados al brand + scope para que el user los pegue en Claude Design.

**Principio de diseño**: cada prompt sigue la estructura best-practice de Claude Design (goal + layout + content + audience) + inyecta context del brand (voice, palette, name, typography) + usa el "design directions" move para que Claude Design proponga opciones antes de build.

## 25.2 Qué es cada prompt

Un "prompt" en nuestra Library NO es un prompt crudo. Es una **plantilla pre-customizada** que:

1. Sigue format Claude Design best-practice
2. Ya tiene el context del brand inyectado (no el user tiene que pegar palette HEX manualmente)
3. Referencia el published design system (Claude Design lo aplica automáticamente after setup)
4. Incluye el "4 design directions" move para iteración

```markdown
# Prompts for Claude Design — {Brand Name}
```

## 25.4 Prompt template — estructura universal

Cada prompt sigue este format:

```markdown
### {Prompt name}

**Goal**: [What you're building]

**Layout**: [How it should be arranged — sections, components, hierarchy]

**Content**: 
- [Specific content item 1 — pulled from our Verbal output when applicable]
- [Specific content item 2]
- [...]

**Audience**: [Who will use it — pulled from target_audience_refined]

**Voice**: Write copy that is {voice attributes}. Reference do/don'ts from brand book:
- Do: {do_example}
- Don't: {dont_example}

**Design**: Apply published design system. {specific design notes if any}.

**Before building**: Propose 4 distinct visual directions tailored to this brief (each as: bg hex / accent hex / approach — one-line rationale), then ask me to pick one and implement only that direction.

---
```

El "Before building" move es crítico — mandates Claude Design to propose options antes de commit a una direction.
