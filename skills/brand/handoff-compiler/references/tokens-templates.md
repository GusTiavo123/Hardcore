# Brand Tokens Templates

Reference consumed by Handoff Compiler in Step 4. Provides templates for each file in `brand-tokens/` folder. Every template is populated with brand-specific values from Visual's palette + typography before being written.

All files in the brand-tokens folder are validated after generation (JSON schema, CSS syntax, Tailwind JS, HTML5 parse). Max 2 retries per file. On persistent failure: include with warning in package README, flag `token_file_validation_failed: {file}` in envelope.

## File 1 — `tokens.css` (CSS custom properties)

Template:

```css
/* {BRAND_NAME} — Design Tokens

 * Import this file after fonts.css in your main stylesheet.
 */

  /* Color — Primary palette */
  --color-primary: {PRIMARY_HEX};
  --color-background: {BG_HEX};
  --color-accent: {ACCENT_HEX};
  --color-text-primary: {TEXT_PRIMARY_HEX};
  --color-text-secondary: {TEXT_SECONDARY_HEX};

  /* Color — Semantic (extend as needed per design system) */
  --color-success: #10B981;
  --color-warning: #F59E0B;
  --color-error: #EF4444;

  /* Typography — Families */
  --font-heading: "{HEADING_FONT}", {HEADING_FALLBACK};
  --font-body: "{BODY_FONT}", {BODY_FALLBACK};
  --font-mono: "{MONO_FONT}", monospace;

  /* Typography — Sizes */
  --font-size-h1: {H1_SIZE};
  --font-size-h2: {H2_SIZE};
  --font-size-h3: {H3_SIZE};
  --font-size-body-large: {BODY_LARGE_SIZE};
  --font-size-body: {BODY_SIZE};
  --font-size-body-small: {BODY_SMALL_SIZE};
  --font-size-meta: {META_SIZE};

  /* Typography — Line heights */
  --line-height-heading: {LINE_HEIGHT_HEADING};
  --line-height-body: {LINE_HEIGHT_BODY};

  /* Typography — Weights */
  --font-weight-heading-default: {HEADING_WEIGHT_DEFAULT};
  --font-weight-body-default: {BODY_WEIGHT_DEFAULT};
  --font-weight-mono-default: {MONO_WEIGHT_DEFAULT};

  /* Spacing (8-point scale) */
  --spacing-xs: 4px;
  --spacing-sm: 8px;
  --spacing-md: 16px;
  --spacing-lg: 24px;
  --spacing-xl: 32px;
  --spacing-2xl: 48px;
  --spacing-3xl: 64px;

  /* Radius */
  --radius-sm: 4px;
  --radius-md: 8px;
  --radius-lg: 16px;
  --radius-pill: 9999px;

  /* Shadows */
  --shadow-sm: 0 1px 2px rgba(0,0,0,0.05);
  --shadow-md: 0 4px 6px rgba(0,0,0,0.08);
  --shadow-lg: 0 10px 20px rgba(0,0,0,0.10);

```

**Fallback stacks** (per font category):
- Serif heading: `"Fraunces", "Playfair Display", "Cormorant", serif`
- Sans heading: `"Inter", "Manrope", "IBM Plex Sans", sans-serif`
- Body: `"Inter", "Manrope", -apple-system, BlinkMacSystemFont, sans-serif`
- Mono: `"JetBrains Mono", "Fira Code", "Courier New", monospace`

## File 4 — `fonts.css` (Google Fonts imports)

Template:

```css
/* {BRAND_NAME} — Font imports
 * Import this file BEFORE tokens.css in your main stylesheet.
 */

@import url('{HEADING_GOOGLE_FONTS_URL}');
@import url('{BODY_GOOGLE_FONTS_URL}');
@import url('{MONO_GOOGLE_FONTS_URL}');
```

**Source**: pull `google_fonts_import` URL from each of Visual's `typography.heading`, `typography.body`, `typography.mono`.

**Consolidation option**: if multiple families can be combined into a single `@import` URL (Google Fonts supports that with `&family=` separators), emit one `@import` line.

## File 5 — `README.md` (inside brand-tokens/)

Template:

```markdown
# {BRAND_NAME} Design Tokens

Design tokens for {BRAND_NAME}. Use in your codebase for consistent brand application.

## Files

- `tokens.css` — CSS custom properties (most flexible)
- `tokens.json` — DTCG format (interoperable across tools)
- `tailwind.config.js` — Tailwind theme extension (drop into a Tailwind project)
- `fonts.css` — Google Fonts imports (must be imported before `tokens.css`)

## Usage — CSS

```html
<link rel="stylesheet" href="fonts.css" />
<link rel="stylesheet" href="tokens.css" />
```

Reference tokens via CSS custom properties:

```css
.button-primary {
  background: var(--color-accent);
  color: var(--color-primary);
  font-family: var(--font-body);
  font-size: var(--font-size-body);
  padding: var(--spacing-sm) var(--spacing-md);
  border-radius: var(--radius-sm);
}
```

## Usage — Tailwind

```bash
# Copy tailwind.config.js to your project
npm install -D tailwindcss
npx tailwindcss -i input.css -o output.css
```

Use Tailwind utilities:
```html
<button class="bg-accent text-primary font-body text-body px-md py-sm rounded-sm">
  Get started
</button>
```

## Usage — Claude Design

Link this folder to Claude Design via codebase linking. Claude Design will auto-extract design system from these files.

## Examples

See the `examples/` folder for rendered component samples using these tokens.

## Regeneration

To regenerate these tokens after brand updates: `/brand:extend handoff`
```

## Critical Rules

1. **Every variable substituted.** No `{VAR}` placeholders remain in output files.
2. **All files validated.** JSON via schema, CSS via syntax check, JS via Node parse, HTML via HTML5 parser. Max 2 retries on failure.
3. **Font fallbacks included.** Never single-family stacks — always include generic fallback (`serif`, `sans-serif`, `monospace`).
4. **Files internally consistent.** HEX in `tokens.css` == HEX in `tokens.json` == HEX in `tailwind.config.js`.
5. **README is actionable.** Users should be able to follow it without external context.
6. **Examples use real brand copy.** Not lorem ipsum. Hero uses real hero headline; card uses one of the brand's value props.
